1. Что такое XML?
	расширяемый язык разметки, созданный для хранения, транспортировки и обмена 
	даннымию. пролог 
2. Чем является первая строка в коде XML-документа? 
	<?xml version="1.0" encoding="UTF-8"?>
	Объявление версии и кодировщика
3. Назовите правила синтаксиса XML-документа?
	1. Все XML элементы должны иметь закрывающий тег.
	2. Теги XML являются регистрозависимыми.
	3. Перед закрывающей угловой скобкой в пустых элементах XML 
	требуется ставить косую черту.
	4. Значения должны быть заключены в одинарные или двойные 
	кавычки.
	5. Все элементы обязаны соблюдать корректную вложенность.
	6. XML документ должен содержать один корневой элемент, который 
	будет родительским для всех других элементов.
	7. Учитываются все символы форматирования (т.е. пробелы, переводы 
	строк, табуляции не игнорируются, как в HTML).
4. Какие должны быть имена элементов?
	Имена могут содержать буквы, числа и другие символы
	Имена не могут начинаться с цифры или символа пунктуации
	Имена не могут начинаться с сочетания "xml" (или XML, или Xml и т.п.)
	Имена не могут содержать пробельные символы
	Элементы могут принимать любые имена
5. Как вы понимаете правило соблюдения корректной вложенности?
	первый открытый тег закрываетя последним, а последний октрытый тег открывается 
	первым
6. Какие элементы являются корневыми в XML-документе?
	Первая строка — это XML декларация. Здесь определяется версия XML (1.0) и 
	кодировка файла. На следующей строке описывается корневой элемент документа 
	(открывающий тег).
8. Для чего необходима валидация XML-документа?
	для проверки синтаксичной верности файла и соответствия определенному 
	типу документов

10. В чем заключается преимущество Schema перед DTD?
	— XML схема пишется на XML;
	— XML схема легко расширяется;
	— XML схема поддерживает типы данных;

11. Какие комплексные типы элементов XML Schema вы знаете?
	Элемент xsd:group
	<xs:sequence> последовательность элементов элемента ;
12. Какие простые типы элементов XML Schema вы знаете?
	<xs:element>	
13. Что означает xs:element?
	строковый тип (текст)
14. Что означает xs:complexType?
	у элемента комплексный тип
15. В чем заключается предназначение xs:sequence?
	последовательность элементов элемента
16. Для чего предназначен xs:attribute?
	Элемент attribute определяет атрибут.
17. Что означает xsi:noNamespaceSchemaLocation?
	Атрибуты noNamespaceSchemaLocation предназначены для 
	подключения XML Schema к XML документу.
	В случае задания схемы без указания пространства имен используется атрибут 
	noNamespaceSchemaLocation.
18. Для чего используется ENTITY?
	для определения сущностей в DTD с целью их использования 
	как в связанном с DTD XML-документе, так и собственно в DTD.
19. Что такое сущности? Каким образом вы использовали сущности?
	Сущность является заместителем содержания, которую можно однажды 
	объявить и многократно использовать почти в любом месте документа.
	
20. Какие встроенные сущности вы знаете?
	lt < (меньше чем) 
	gt > (больше чем)
	amp & (амперсанд)
	apos ' (апостроф или одиночная кавычка)
	quot " (двойная кавычка)
21. Для чего используется ATTLIST?
	для перечисления и объявления атрибутов, которые могут 
	принадлежать элементу
22. Что такое валидный XML-документ? Как типы вы знаете?
	Валидный XML-документ должен быть синтаксически верным и 
	соответствовать одному из типов определения документов. Правила, 
	определяющие допустимые элементы и атрибуты для XML документа, 
	называются определениями документа или схемами документа.
	SML DTD XSD
23. Что означает следующая запись <!ENTITY name “Hello, world!”>?
	переменная name со значением “Hello, world!”
24. Что означает следующая запись <!ELEMENT to (#PCDATA)>?
	определяет, что элемент to должен быть 
	типа "#PCDATA"

26. Какие параметры имеет инструкция ELEMENT?
